# Lessons Learned — Structured Reference

This file is a decision aid for future Hew work. It is not a changelog.

Use it in this order:

1. Match the current change against a **core invariant** in the matching priority band.
2. Walk the **nuances** under that core; the trigger column tells you which apply.
3. Treat the **apply** column on each matching nuance as the minimum pre-merge checklist.
4. Start with the **P0** cores before applying any matching **P1** or **P2** nuances.
5. When two rules conflict, keep the stricter fail-closed, ownership-preserving, or parity-preserving rule.

## Two-level schema

A **core invariant** is the durable rule a future agent should remember even if every
nuance below it is forgotten. A **nuance** is a specific application, smell, or bug
class that has been seen in this codebase. Multiple nuances usually share one core.

Core block:

| Field | Meaning |
| --- | --- |
| `core` | Stable short key for the core invariant. |
| `area` | Subsystem(s) where the core fires. |
| `invariant` | One-sentence durable rule. |
| `why` | The failure class the core prevents. |

Nuance row (table under the core):

| Column | Meaning |
| --- | --- |
| `id` | Stable short key. **External code, docs, and specs reference these by string** — never rename a nuance id. |
| `trigger` | Smell or change pattern that should activate the nuance. |
| `apply` | Concrete check or action to take now. |
| `evidence` | Main PRs / JOURNEY workstreams that produced the nuance. |

## Priority bands

| Band | Use it for |
| --- | --- |
| `P0` | Silent wrong-code, memory safety, ownership, cleanup, or cross-language boundary risk. |
| `P1` | Parity, tests, diagnostics, toolchain, or user-surface trust. |
| `P2` | Architecture, cleanup, deduplication, and maintenance decisions that prevent repeat bugs. |

## P0 — Compiler/runtime correctness and boundary safety

### `boundary-fail-closed` — every cross-boundary handoff either carries the full payload or stops

- **area**: serialization, codegen, FFI, distributed surface, generated code
- **invariant**: A boundary that crosses Rust↔C++, host↔runtime, local↔remote, or generator↔repo must encode its full supported form or fail loudly with context. Silent omission, sentinel-shaped success, and emitted-but-uncounted diagnostics are bugs.
- **why**: Boundaries are where types lose their bite. Anything dropped or coerced to a sentinel becomes a silent wrong-output channel that the type system can no longer protect.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `serializer-fail-closed` | A Rust-to-C++ or wire boundary must encode types, schema data, or unsupported shapes. | Reject unsupported nested forms, carry span/context through conversion, turn warning-and-skip paths into hard errors, and version or contract-test high-risk seams. | PR #812/#811/#808/#795. |

Both are v0.5.1 per-field-attribution follow-ons. **W5.021 revision 2 (owned handle-place members, 2026-06-03):** the value-flow member derivation must admit the SAME `Place` set on the producer side (the `ReturnSlot` seed, the `add_member` source predicate, the `TupleConstruct`/`RecordInit` decomposition) that the map-back side resolves via `base_local`. The asymmetry is the smell: when a value-flow exclusion pass resolves a binding's local through a place-normalising helper (`base_local`) on one side, every seed/propagation predicate on the other side must accept the identical place set, or any place that helper normalises-but-the-predicate-rejects falls through the hole. It is the INVERSE of `place_is_interior_projection`: that classifier over-taints handle places (fail-closed for ITS purpose, over-tainting only over-excludes), this one accepts them (fail-closed for OURS, accepting a hand-off slot is exactly what stops the double-drop). ### `checker-authority` — checker output is authoritative, downstream consumes, no re-inference

- **area**: typecheck, enrichment, serialization, codegen
- **invariant**: Whatever the checker resolves — receivers, types, generics, visibility, assignment targets, ABI choices, pattern legality — must survive the boundary into serialization and codegen. Downstream phases consume those facts; they do not rediscover them by AST fallback or span-keyed lookup.
- **why**: Re-inference at codegen time is racy with checker order and stale across function scopes, and span-keyed lookups can be absent or contradict metadata the lowering pass already tracked. Once an authority loses its monopoly, two phases disagree and silently produce wrong code.

### `exhaustive-coverage` — no wildcard fallthrough in dispatch or lowering

- **area**: analysis, codegen, match
- **invariant**: Visitors, dispatch chains, and match lowerings must enumerate every supported variant and every operation×type pair. Wildcard or default branches in semantic positions hide gaps as silent wrong code; cross-enum fallback and guarded wildcards count as gaps.

- **area**: control flow, codegen RAII, FFI handles, runtime shutdown, concurrent test cleanup

## P2 — Architecture, cleanup, and maintenance decisions

### `dedup-with-policy-boundary` — share mechanics, stop deduplication where semantics diverge

- **area**: parser, runtime helpers, type-checker traversals
- **invariant**: Helpers that look identical but diverge at one semantic edge should share scaffolding (parse, decode, structural recursion) but keep per-owner policy at the call site. Centralise the walk; do not centralise the decision.
- **why**: Premature unification flattens away exactly the policy that mattered, then re-grows it as flags or branches that recreate the duplication with worse cohesion.

### `architecture-cleanup-discipline` — scope decisions before structure decisions

- **area**: cleanup sweeps, integration vs rewrite, generator ceilings, lockfile resolution
- **invariant**: Before deleting an apparently-dead surface, re-classify it across docs/tests/specs/completions/workspace. Before rewriting a working module, prefer a thin integration owner over new abstraction layers. Generators have a ceiling — they emit mechanics, not semantics — and resolution algorithms must be restartable, not single-pass.
- **why**: "It looks unused" misses non-production callers; "let's rewrite this" loses working invariants; "let's autogenerate this" produces façades that look right but encode the wrong public name or return type. The cheap path is structural; the right path is semantic.

### `migration-completeness` — a replacement is not done until the predecessor is gone or fails loud

- **area**: runtime ABI/symbol families, codegen lowering, checker dispatch tables, IR-ladder lowering, serialization
- **invariant**: When you introduce a new authority for a decision (a symbol family, an ABI, a resolver, a lowering arm, a type carrier, a dispatch table), the SAME change must either delete the predecessor or make every remaining caller of it fail loud (`unreachable!`/diagnostic). Two live authorities for one decision is a defect, not a milestone — never "land the new path, retire the old one later."
- **why**: A superseded-but-reachable path silently serves stale data, disagrees with the new authority, and degrades into wrong code or memory corruption. Every parallel authority left alive is a future UAF, ABI mismatch, or fail-open hole waiting for the wrong caller to reach it. This is the P0 `checker-authority` failure generalised to every layer: once an authority loses its monopoly, two paths disagree and silently produce wrong output.

| --- | --- | --- | --- |
| `retire-in-lane-or-fail-loud` | A change adds a superseding implementation (e.g. a `_layout` symbol family, a typed handoff, a descriptor-driven resolver) while the legacy path still compiles and is reachable. | Retire the predecessor in the SAME lane. If retirement must be staged (real downstream caller can't migrate yet), make the legacy entrypoint `unreachable!("<reason>")` or emit a diagnostic so exercising it fails noisily, AND file the retirement as a tracked, gated lane. Never leave both paths silently callable. | W4.045: HashSet drop/clone kept wiring legacy `hew_hashset_free`/`hew_hashset_clone` onto `hew_hashset_new_with_layout` handles after the `_layout` migration → 48-vs-16-byte stride → `libc::free(garbage)` heap-corruption UAF. ### `actor-surface-no-ask-verb` — actor request-reply is signature-driven, not keyword-driven

- **area**: language surface, actor model, spec review
- **invariant**: Hew's actor call surface is signature-driven dispatch. A `receive fn` returning `R` is a request-reply; one returning `()` is fire-and-forget. No caller-side keyword (`ask`, `.ask()`, `!`, `?`) is needed or permitted. `ask` is not lexer-recognised at any position in edition 2026; the select-arm actor discriminator is shape-based (`<actor>.<method>(<args>)` at HIR lowering), not a keyword (§4.11.1).
- **why**: Pre-2026 editions mirrored Akka classic (`.send()`/`.ask()`). The 2026 rewrite eliminated both: the checker derives tell-vs-ask from the callee's `receive fn` return type, and `ActorRef<T>` is already a distinct nominal type that marks the mailbox boundary. Reintroducing a verb would regress per-method type checking and break the signature-driven invariant without adding expressiveness.

### `structured-concurrency-scope-fork-split` — `scope` is the lifetime boundary; `fork` is the child-start verb

- **area**: language surface, structured concurrency, parser/HIR/codegen seam, spec review
- **invariant**: Hew's structured-concurrency surface is two non-synonymous keywords. `scope { ... }` opens the lexical-lifetime boundary; child tasks spawned inside are joined automatically at the closing brace. `fork name = call(...)` (or bare `fork call(...)`) inside a scope body is the only child-start verb; it is rejected outside a scope. The legacy `scope |s| { s.launch / s.spawn / s.cancel }` surface and the `fork { ... }` block form are both removed and must not be reintroduced; lifecycle hooks attach via actor-method attributes (e.g. `#[on(start)]` / `#[on(stop)]`), not via scope methods.
- **why**: Early Hew conflated the lifetime boundary with a lifecycle-method API (`scope |s|` with `s.launch / s.spawn / s.cancel`). Type-checking, HIR lowering, and C++ codegen all carried the cost: an extra parser state field, a desugar branch in `parse_expr`, three dead AST/IR variants, and a C++ MLIR op pair (`ScopeLaunchOp` / `ScopeCancelOp`) that only the legacy surface emitted. The 2026 ratification removed it entirely (no external consumers; in-tree migration). Reviving any part of it — a `fork { ... }` block form, an `s.launch / s.spawn` method, a unified `fork-is-also-scope` keyword — would re-conflate the two roles and reintroduce the seam.
