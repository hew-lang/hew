# Hew v0.5.0

Hew v0.5.0 is the first release in which the language, runtime, and
toolchain stand on a single coherent substrate that distributed-systems
engineers can build on without reaching past the C ABI for the
load-bearing parts. Service backends, real-time pipelines, and the
small fleet of long-lived processes that sit behind them are the
programs this version of Hew is designed to carry.

## Why v0.5.0 matters

v0.3 made Hew usable. v0.4 made the language durable: explicit
resource ownership, a daily-use toolchain, a non-crashing JIT, and a
standalone release binary. Those two releases bought the right to do
this one.

v0.5 consolidates the **substrate**. The pieces that distributed
backend engineers reach for first — typed channels and structured
concurrency, supervised actors with crash/upgrade hooks, a named
distribution layer with mutual-TLS transport, fail-closed integer and
slice semantics, a wire-format doctrine that draws a hard line between
the runtime envelope and user-visible serialisation, and a sandbox VM
that admits the same programs the native backend admits — are now in
the box and aligned with each other. None of these are individually
new ideas in the field. What is new is that they share a single
internal vocabulary in this codebase: the typed HIR/MIR/codegen-rs
ladder lowers them all through the same checked path, and the trust
bar (LSP, formatter, parity harness) follows them through.

That alignment is what justifies a minor-version step rather than a
patch-level accumulation of fixes. The breaking changes that ship
with this release retire the v0.4-era scaffolding that the new
substrate replaces; the migration paths are caught at compile time
with fixit notes.

## From v0.4 to v0.5

v0.4 said: Hew code reads like Hew code, and the seams to C are
visible exactly where they exist. v0.5 follows through by **raising
the load-bearing surfaces out of the C ABI** and into Hew, one rung
at a time.

The release is organised around six themes:

1. The **substrate ladder L0–L3** — `extern`/`unsafe`,
   `Result`/`Option` constructors, raised string methods, `Display`
   and f-string interpolation.
2. The **native mesh** — `Node::*`, `RemotePid<T>`, `LookupError`,
   and mTLS over QUIC with SPKI-pinned identity.
3. The **actor-first runtime** — link/monitor/exit, supervisor
   surface, lifecycle hooks, `max_heap`, and partition signalling.
4. **Primitive and record correctness** — explicit integer widths,
   fail-closed traps, named-field records with functional update.
5. **Sandbox parity** — a single canonical frontend for native and
   sandbox bytecode, with a parity harness that fails closed.
6. **The wire-format doctrine** — CDDL+CBOR for the runtime envelope,
   `std::encoding::*` for user code, and an explicit anti-doctrine
   for everything else.

A seventh theme — **developer trust** — runs through all six: the
WASM-LSP analyzer, the browser-LSP compile path, the editor grammar
bundles, and the diagnostic and observability surfaces are tightened
alongside the substrate they describe.

## What you can now rely on

The v0.5.0 release commits to the following invariants. Code that
respects them will not be quietly broken by point releases on the
v0.5 line.

- **The substrate ladder is closed.** `extern` and `unsafe` are the
  only admission points for foreign or authority-bearing operations.
  Above that line, the language carries its own constructors, its own
  string and slice operations, and its own formatting trait.
- **Channels and actors share one method surface.** `.send`, `.recv`,
  `.try_send`, `.try_recv`, `.close`, `.send_half`, `.recv_half` mean
  the same thing on a `Duplex`, a `Sink`, a `Stream`, and a
  lambda-actor handle.
- **Structured concurrency is enforced.** A `scope` block does not
  return until its children complete; a fault in one child cancels
  its siblings. There is no path that leaks an unjoined task past
  its lexical scope.
- **Supervision is the default failure posture.** Actor faults are
  caught by the supervisor and observable through `#[on(crash)]`
  with structured `CrashInfo` and `TrapInfo`. Crashing an actor does
  not crash the process.
- **Integer arithmetic fails closed.** Overflow, divide-by-zero,
  modulo-by-zero, and shift-out-of-range trap with a typed
  diagnostic. Wrapping behaviour is opt-in on every operation that
  needs it.
- **Slice and vector access fails closed.** `Vec<T>` indexing and
  range slicing trap on out-of-bounds rather than returning garbage.
- **The native mesh has typed identity.** `RemotePid<T>` names a
  remote actor, `LookupError` enumerates lookup failures, and node
  connections refuse anonymous inbound peers.
- **The inter-process wire format is fixed.** CDDL-schema CBOR
  envelopes with explicit `version` rejection. User code does not
  reach into the envelope.
- **The sandbox and native backends share their frontend.** A
  program that admits on one admits on the other, modulo the
  divergences explicitly catalogued in
  [`docs/sandbox-vm-divergences.md`](../sandbox-vm-divergences.md).

## The substrate ladder, L0 through L3

The core idea of v0.5 is the **substrate ladder**: a small number of
rungs that each take a load-bearing piece of the language out of the
opaque builtin or extern-bridge bucket and put it on the same checked
ladder that user code uses. The ladder is read bottom-up.

### L0 — `extern` and `unsafe` are the admission line

Foreign-function calls, raw pointer manipulation, and any operation
that requires authority the type system cannot vouch for must live
inside `extern` or `unsafe` blocks. The checker refuses to admit
those operations outside the gate. This is the boundary against
which every higher rung is built: once L0 is in place, the upper
rungs do not need to re-litigate authority — they only need to live
above the line.

### L1 — `Result` and `Option` constructors

`Ok(x)`, `Err(e)`, `Some(x)`, and `None` are first-class Hew
constructors in v0.5, not opaque builtin shims. Their match-arm
patterns lower through the canonical HIR enum path. Two things
become possible because of this. First, the `?` operator and the
new `let r? = expr` sugar do not need a special-case codepath — they
desugar through the same enum-match lowering everything else uses.
Second, user code can pattern-match on the `Result`/`Option`
constructors at the same surface where library code produces them,
with no semantic seam between them.

### L2 — Strings are first-class

The methods that programmers actually call on strings — `.len()`,
`.slice(a, b)`, `.find(needle)`, indexing, the `+` concatenation
operator — are now ordinary Hew methods. They operate on Unicode
codepoints, not raw bytes, and they desugar through the same method
surface as user-defined extension traits. The previous bridge to a
runtime helper is gone for these operations.

This rung matters in two places that bite distributed-systems
programs the most: parsing wire input and producing human-readable
diagnostics. Both are now written against a string surface that the
type checker understands at the same level as everything else.

### L3 — `Display` trait and f-string interpolation

`Display` is the user-facing formatting trait. `f"the value is {x}"`
lowers to `x.display(...)` against a `Formatter` argument rather
than to an opaque builtin format builtin. User types — record types,
enum variants, the actor identity types — participate in formatting
by implementing `Display`. The same f-string surface programmers use
for log lines is the surface library code uses for diagnostic
messages.

Together the four rungs mean a v0.5 program can be written
end-to-end — input parsing, business logic, error handling, log
output — without reaching through an extern bridge for any of its
common operations.

## The native mesh substrate

Hew v0.5 lands the first version of the **native mesh**: a
language-layer surface for actors that live on different processes
and different machines.

### The `Node::` namespace

The mesh is exposed at the language layer as a `Node::` namespace
with six entry points:

- `Node::start(config)` — bring a node up.
- `Node::connect(peer)` — connect this node to a named peer.
- `Node::shutdown()` — drain in-flight messages and tear the node
  down.
- `Node::set_transport(t)` — install a transport (the default is
  QUIC with mTLS, described below).
- `Node::register(name, actor)` — publish a local actor under a
  registry name.
- `Node::lookup(node, name)` — resolve a registered name on a peer
  node to a typed handle.

The names are deliberate. The mesh is not a hidden background — it
is named code that a programmer reads and writes. A program that
ignores the mesh does not pay for it; a program that needs the mesh
calls into a namespace that the type system understands.

### `RemotePid<T>` and `LookupError`

The split between local and remote actor identity is now reflected
in the type system. `LocalPid<T>` names a local actor;
`RemotePid<T>` names an actor on a peer node. The same-node
fast path and the cross-node serialisation path are different code
paths inside the runtime, and forcing them to be different at the
type level keeps the user out of the failure modes that come from
treating them as one.

`Node::lookup` returns `Result<RemotePid<T>, LookupError>`, where
`LookupError` is a typed sum: `NotFound`, `WrongType`,
`NodeUnreachable`, and friends. Stringly-typed lookup errors are
gone from the user surface.

### mTLS over QUIC with SPKI pinning

The default mesh transport is QUIC with mutual TLS. Peer identity
is pinned by an SPKI (subject-public-key-info) allowlist rather
than CA trust: a node accepts an inbound peer only if the peer's
public-key fingerprint appears in the configured allowlist. The
handshake refuses downgrade attempts, version-mismatch attempts,
and identity-mismatch attempts; anonymous inbound mesh connections
are refused outright.

This is not the same as full PKI infrastructure, and it is not
trying to be. The choice is deliberate: distributed-systems
deployments at the scale Hew is built for typically configure a
small, known set of peers, and SPKI pinning is the smallest trust
surface that captures the actual deployment. Code that wants
CA-rooted identity can install a different transport via
`Node::set_transport`.

### A two-node cross-process integration test

A dedicated integration test starts two Hew processes on the local
machine, registers an actor on node A, looks it up from node B, and
runs an ask round-trip across the mesh. The test covers the
register → lookup → typed-handle → ask → reply sequence
end-to-end, with both nodes in real subprocesses and the QUIC mTLS
handshake live. It is the regression backstop for the mesh surface.

## The actor-first runtime

The runtime that sits underneath the mesh is reorganised around
supervised actors as the primary unit of concurrency. The pieces
distributed-systems programmers reach for are now part of the
language layer.

### Link, monitor, exit

`link(other)` establishes a bidirectional liveness link between two
actors: when one dies, the other receives an exit signal. The
default exit-signal handling is to propagate — a linked sibling's
crash crashes the linked actor, which the supervisor then catches.
`monitor(other)` is the one-directional variant: the monitoring
actor receives a notification when the monitored actor exits, but
the monitored actor does not learn about the monitor. Exit signals
carry the structured `CrashInfo` described below.

### Supervisor surface

Supervisors declare their restart strategy alongside their
supervised children. `one_for_one` restarts only the failing
child. `one_for_all` restarts the whole supervised group. `rest_for_one`
restarts the failing child and every child started after it. The
strategies match the established failure-philosophy invariants
(spec §5.17): siblings are isolated fault domains, faults are
caught at the supervisor and not at the language-level call site,
and the default posture is restart rather than abort.

### Lifecycle hooks — `#[on(start)]`, `#[on(stop)]`, `#[on(crash)]`, `#[on(upgrade)]`

Actor methods carry lifecycle annotations:

- `#[on(start)]` runs before the message loop begins. At most one
  per actor.
- `#[on(stop)]` runs after the loop exits, whether by normal stop
  or supervised termination. Multiple are allowed and execute in
  lexical declaration order.
- `#[on(crash)]` runs on the supervisor side when a child faults.
  At most one per actor. Receives the structured `CrashInfo`.
- `#[on(upgrade)]` runs when a hot-replaced actor body is installed
  over a live mailbox. At most one per actor.

The `terminate { ... }` block syntax that v0.4 accepted is gone;
its semantics are folded into `#[on(stop)]`.

### `max_heap` per-dispatch arena cap

Each actor declares a `max_heap` cap. Exceeding it triggers a
supervised restart rather than an OOM-killed process. The default
is generous, but the existence of the cap closes the failure mode
in which a single misbehaving actor pulls down its entire host
process. Combined with structured supervision, this means a Hew
backend's worst case for a runaway allocation is a restarted
actor, not a restarted process.

**Scope clarification**: `max_heap` is enforced **per-dispatch
arena residency**. The arena is reset between handler invocations,
so the cap bounds the memory live at any one moment within a single
dispatch cycle — not the actor's total allocation over its
lifetime. Per-actor-lifetime growth (mailbox queue, state record,
etc.) is not bounded by this cap. If you need a per-actor-lifetime
memory budget, pair a mailbox-depth bound with an explicit
state-size budget; a dedicated mechanism is tracked as a v0.5.1
backlog item.

### `PartitionDetected` and the removal of `Blocked`

When the local node loses connectivity to a peer, monitors of
remote actors on that peer receive a
`PartitionDetected { node, last_seen }` notification distinct from
ordinary actor-exit signals. Code that needs to behave differently
under partition — for example, by waiting rather than restarting —
can pattern-match on the partition signal.

The internal `Blocked` actor state is removed. The scheduler now
models awaits and blocking I/O through receive-cursor state on the
actor's mailbox and scope, rather than through a distinct state
machine entry. The change simplifies both the scheduler and the
traces it emits — the v0.5 concurrency-event taxonomy described
below benefits directly.

## Primitives and records

The third theme is correctness in the small. Distributed-systems
programs spend a lot of time threading integers through wire
boundaries and packaging records into and out of envelopes. v0.5
fixes the failure modes that bite that work.

### Integer width canonicalisation

The integer surface is explicit: `i8`, `i16`, `i32`, `i64`, `u8`,
`u16`, `u32`, `u64` for fixed-width integers; `isize` and `usize`
for pointer-sized integers. The v0.4-era `int` and `uint` aliases
are gone — the compiler emits a diagnostic with the correct
suggestion when those names appear in type annotations. The
removal is deliberate: at the wire boundary, "the natural integer"
is not a meaningful concept; the program needs to say which width
it means.

Untyped integer literals inherit context width with a defined
defaulting rule, so the cost of the explicit surface is paid only
where it actually matters.

### Fail-closed integer traps

Overflow, divide-by-zero, modulo-by-zero, and shift-out-of-range
now **trap** with a typed `TrapInfo`. Programs that genuinely want
wrapping behaviour opt in on each operation:
`.wrapping_add(...)`, `.wrapping_sub(...)`, `.wrapping_mul(...)`,
`.wrapping_shl(...)`, `.wrapping_shr(...)`. The wrapping methods
are available on every integer width and on `isize`/`usize`.

This is a behaviour change from v0.4 and earlier. Programs that
relied on silent wraparound need to either accept the trap and fix
the underlying logic, or convert to the `.wrapping_*` form
explicitly. The trade-off is intentional: at a wire boundary, a
silent wraparound is almost always a bug, and the cost of catching
it at runtime is small compared to the cost of debugging it in
production.

### Vec bounds, range slice

`Vec<T>` indexing traps on out-of-bounds access rather than
returning garbage or silently masking. `vec.slice(a, b)` returns a
borrowed range slice; an OOB range traps with a typed diagnostic.
The same pattern that applies to integer arithmetic applies here:
silent failure at a wire boundary is the more dangerous failure
mode.

### Record types

Record types are the v0.5 successor to the v0.4 struct-shaped
data:

- **Named-field declaration.** `record Point { x: f64, y: f64 }`.
- **Brace literals.** `Point { x: 3.0, y: 4.0 }`.
- **Auto-derive of `Eq` and `Hash`.** A record satisfies `Eq` and
  `Hash` when all of its fields do; using a record as a `HashMap`
  key without that closure is a compile error rather than a silent
  runtime failure.
- **Functional update.** `Point { x: 3.0, ..base }` produces a
  copy of `base` with the listed fields overridden.
- **Tuple-record form.** A positional-field record form is also
  accepted for cases where the field names would be noise.
- **The `is` operator.** `value is Point` performs a structural
  shape check against either form of record.

The combination — explicit named-field types with auto-derived
equality, functional update, and a structural shape check — is the
v0.5 vocabulary for the data envelopes that flow through wire
boundaries.

## Sandbox runtime parity

The fourth theme is **parity** between the sandbox bytecode VM (the
deterministic, replay-friendly runtime that hosts Hew programs in
the browser sandbox and other constrained execution environments)
and the native LLVM backend.

### One canonical frontend

The sandbox bytecode emitter in v0.5 goes through the canonical
typed-IR frontend rather than a separate AST walker. From source
text through MIR, the sandbox path and the native LLVM path share
their lowering. A bug fixed in one is fixed in both; a program that
admits on one admits on the other.

### Control-flow lowering parity

`match`, `if`/`else`, `while`, `for`, `?`-propagation,
`scope`/`fork`, and `gen` blocks all share control-flow shape
between native and sandbox targets. The control-flow surface that
the type checker exposes to the user is the same surface both
backends consume.

### `make sandbox-parity`

A dedicated make target — `make sandbox-parity` — runs every parity
fixture through both backends, diffs observable outputs, and fails
closed on divergence. New tests that exercise observable behaviour
should run through this harness; tests that need to behave
differently on the two backends must be admitted through the
divergence catalog.

### The divergence catalog

The accepted-divergence catalog lives at
[`docs/sandbox-vm-divergences.md`](../sandbox-vm-divergences.md).
The catalog enumerates the divergences that are deliberate —
deterministic scheduling, virtual clock, seeded PRNG, logical heap
accounting, deterministic actor and message IDs, page-backed
stdout/stderr/stdin, in-memory-only streams, single-threaded Web
Worker execution, regex-engine differences — and the surfaces that
are explicitly out of scope for the sandbox (host filesystem,
host network, host clock, host entropy). New divergences must be
added to the catalog with a reason.

## The wire-format doctrine

The fifth theme is the **wire-format doctrine** —
[`docs/specs/HEW-WIRE-FORMAT-DOCTRINE.md`](../specs/HEW-WIRE-FORMAT-DOCTRINE.md)
is the doc-of-truth. The doctrine is short, and the short form is
this: different layers of the system have different wire-format
needs, and v0.5 fixes which format is canonical at which layer.

### CDDL + CBOR for inter-process messaging

The runtime's actor envelope is a CBOR `wire-frame` keyed by small
integers (1..=8) and described by a CDDL schema at
`hew-runtime/schemas/envelope.cddl`. An envelope frame carries
`version`, `frame_type`, `target_actor_id`, `source_actor_id`,
`msg_type`, `payload` (the user-level body as bytes), `request_id`,
and `source_node_id`.

The `version` field is currently `1`. Decoders **must** reject
unknown versions with `UnknownVersion`; they must not best-effort
parse a frame whose version they do not recognise. This is the
fail-closed rule that allows the envelope format to evolve in
later versions without ambiguous decoding behaviour.

### `std::encoding::*` for user code

Programs that need to read or write wire bytes at the language
layer use the standard-library encoding modules: `std::encoding::json`,
`std::encoding::cbor`, and friends. The runtime envelope is not a
user-extension surface — user code does not reach into it. The
boundary between "the runtime moves messages between actors" and
"the program reads and writes external wire bytes" is explicit.

### Anti-doctrine

Three patterns are explicitly out-of-scope for v0.5:

- **Schema-first consumer-interop generators** (OpenAPI,
  `proto-gen`, and similar) are deferred to v0.5.1 and beyond.
- **Ad-hoc msgpack frames** in user code are not supported by the
  runtime envelope path; the `legacy-wire-msgpack` Cargo feature
  is gone.
- **Reaching into the runtime envelope** from user code is not a
  supported extension point. The envelope is owned by the runtime
  and the migration commitments around it are made there.

The full rationale, including the comparison against JSON, msgpack,
Protobuf, and the historical HBF audit, is in the doctrine
document.

## Developer trust

The sixth theme threads through all five of the above. The
toolchain has to follow the substrate; otherwise the substrate is
not actually usable.

### WASM-LSP analyzer fixtures

The WASM-hosted language-server analyzer ships with thirty new
v0.5-substrate fixtures. The fixtures exercise hover,
goto-definition, find-references, signature help, and diagnostics
against the new substrate — channel types, lambda-actor handles,
`scope`/`fork` blocks, lifecycle hooks, record literals with
functional update, the new integer surface, and the `RemotePid<T>`
identity type.

### Browser-LSP compile path

The browser-LSP compile path — parse → check → diagnostic
emission — is covered alongside the analyzer fixtures. A program
that admits in the editor admits in the compiler, and a program
that produces a diagnostic in the compiler produces the same
diagnostic in the editor.

### Editor grammar keyword updates

The Sublime, Emacs, and Nano syntax bundles are updated for the
v0.5 keyword set: `actor`, `scope`, `fork`, `gen`, `#[on(...)]`,
`record`, `isize`, `usize`, and the removed-keyword diagnostics for
`terminate`, `spawn`-lambda syntax, `int`/`uint`, and the legacy
`<-` operator. Editor users see the same keyword set the compiler
admits.

## Diagnostics and observability

The diagnostic and observability surfaces follow the substrate.

### `CrashInfo` and `TrapInfo`

When a supervisor catches a child fault, the structured
`CrashInfo` is available to the `#[on(crash)]` handler: `reason`,
`source_location`, `actor_id`, `restart_count`. The underlying
`TrapInfo` — trap kind, faulting instruction, backtrace — is
attached. The supervisor sees the structured failure rather than
an opaque string, which means restart decisions can depend on the
failure type rather than parsing diagnostic text.

### Profiler schema versioning

Profiler output carries an explicit `schema_version` field. Older
trace consumers that do not recognise the v0.5 schema receive a
typed mismatch error rather than silently drifting against a
moved schema. The doctrine here is the same as the wire-format
doctrine's `version` rule: decoders that do not recognise a
version fail closed.

### Trace taxonomy v0.5

The v0.5 concurrency-event taxonomy emits structured events for
`fork`, `join`, `cancel`, `ask`, `reply`, `partition`, `restart`,
and `upgrade`. The taxonomy matches the language-level surface that
produces the events, which makes the traces interpretable at the
level the program was written at.

### Actor `Blocked` state removed from traces

With the internal `Blocked` actor state gone from the scheduler,
the traces no longer carry the spurious transitions through it.
Await and blocking-I/O behaviour shows up in the trace as
mailbox-and-scope state, which matches the model the language
exposes.

## What's next — v0.5.1

The v0.5.1 work is the **stdlib polish** that the substrate now
admits. Three pieces are committed for the next minor:

- **L4 — idiomatic stdlib rewrite.** With the substrate ladder
  closed at L0–L3, the standard library is rewritten to live above
  the ladder rather than alongside it. The rewrite is mostly a
  surface change — the underlying runtime behaviour does not move —
  but it closes the gap between "what user code looks like" and
  "what stdlib code looks like."
- **Trait-bound machine generics + `Lifecycle<T>`.** The machine
  surface gains trait-bound generics, with `Lifecycle<T>` as the
  first standard machine trait. The current associated-types
  surface (one associated type per trait) is the foundation this
  work builds on.
- **`MethodCallRewrite` → HIR/MIR bridge cleanup.** The remaining
  method-call rewrites that still pass through a bridge layer
  outside the typed IR ladder are moved onto the canonical HIR/MIR
  path. The change closes the last gap between "method call as the
  user wrote it" and "method call as the type checker and codegen
  see it."

Beyond v0.5.1, the **L5 ladder rung** — generic actor-handle
inference and supervised-generic-actor inference — is the next
substrate-level rung. It is named here for context; specific
commitments will land in the v0.6 release notes when its
prerequisites have settled.

v0.5 is the largest release the project has shipped: it consolidates
two years of language-level and runtime-level work into one
substrate that programmers can build on. The breaking changes that
ship with it are the cost of getting that consolidation right the
first time, and every removed surface has a fixit-noted migration
in place. The companion `CHANGELOG.md` entry for `[0.5.0]` is the
