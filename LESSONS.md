# Lessons Learned — Structured Reference

This file is a decision aid for future Hew work. It is not a changelog.

Use it in this order:

1. Match the current change against a **core invariant** in the matching priority band.
2. Walk the **nuances** under that core; the trigger column tells you which apply.
3. Treat the **apply** column on each matching nuance as the minimum pre-merge checklist.
4. Start with the **P0** cores before applying any matching **P1** or **P2** nuances.
5. When two rules conflict, keep the stricter fail-closed, ownership-preserving, or parity-preserving rule.

## Two-level schema

A **core invariant** is the durable rule a future agent should remember even if every
nuance below it is forgotten. A **nuance** is a specific application, smell, or bug
class that has been seen in this codebase. Multiple nuances usually share one core.

Core block:

| Field | Meaning |
| --- | --- |
| `core` | Stable short key for the core invariant. |
| `area` | Subsystem(s) where the core fires. |
| `invariant` | One-sentence durable rule. |
| `why` | The failure class the core prevents. |

Nuance row (table under the core):

| Column | Meaning |
| --- | --- |
| `id` | Stable short key. **External code, docs, and specs reference these by string** — never rename a nuance id. |
| `trigger` | Smell or change pattern that should activate the nuance. |
| `apply` | Concrete check or action to take now. |
| `evidence` | Main PRs / JOURNEY workstreams that produced the nuance. |

## Priority bands

| Band | Use it for |
| --- | --- |
| `P0` | Silent wrong-code, memory safety, ownership, cleanup, or cross-language boundary risk. |
| `P1` | Parity, tests, diagnostics, toolchain, or user-surface trust. |
| `P2` | Architecture, cleanup, deduplication, and maintenance decisions that prevent repeat bugs. |

## P0 — Compiler/runtime correctness and boundary safety

### `boundary-fail-closed` — every cross-boundary handoff either carries the full payload or stops

- **area**: serialization, codegen, FFI, distributed surface, generated code
- **invariant**: A boundary that crosses Rust↔C++, host↔runtime, local↔remote, or generator↔repo must encode its full supported form or fail loudly with context. Silent omission, sentinel-shaped success, and emitted-but-uncounted diagnostics are bugs.
- **why**: Boundaries are where types lose their bite. Anything dropped or coerced to a sentinel becomes a silent wrong-output channel that the type system can no longer protect.

### `checker-authority` — checker output is authoritative, downstream consumes, no re-inference

- **area**: typecheck, enrichment, serialization, codegen
- **invariant**: Whatever the checker resolves — receivers, types, generics, visibility, assignment targets, ABI choices, pattern legality — must survive the boundary into serialization and codegen. Downstream phases consume those facts; they do not rediscover them by AST fallback or span-keyed lookup.
- **why**: Re-inference at codegen time is racy with checker order and stale across function scopes, and span-keyed lookups can be absent or contradict metadata the lowering pass already tracked. Once an authority loses its monopoly, two phases disagree and silently produce wrong code.

| id | trigger | apply | evidence |

### `exhaustive-coverage` — no wildcard fallthrough in dispatch or lowering

- **area**: analysis, codegen, match
- **invariant**: Visitors, dispatch chains, and match lowerings must enumerate every supported variant and every operation×type pair. Wildcard or default branches in semantic positions hide gaps as silent wrong code; cross-enum fallback and guarded wildcards count as gaps.
- **why**: Adding a new AST variant or operand type without an exhaustive matrix lets the new shape silently take the default arm. The compiler keeps producing output, but the output is wrong for the new case.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `exhaustive-traversal-and-lowering` | A new AST variant, lowering branch, or supported type is added. | Maintain explicit variant/type lists, build an operation × type matrix for typed lowerings, cover narrow widths and uncommon shapes, and prefer errors over warnings when a case is unsupported. | PR #798/#786/#797. |
| `match-fail-closed` | Match checking or lowering changes. | Forbid cross-enum fallback, treat guarded wildcards as non-exhaustive, exclude `Never` and `Error` from result inference, wrap `HashMap.get()` at the expression boundary, trap on runtime fallthrough, and emit `MirStatement::Bind` (not just `Instr::Move`) at the entry of every arm body that introduces a payload binding — the move-checker dataflow keys on `Bind` to lift the binding from `Uninit` to `Live`, so a bare `Move` into the binding's local trips `InitialisedBeforeUse` on the first arm-body read. | Original match correctness audit; `feat/mono-mixed-enum-payload-ctor`. |

### `lifecycle-symmetry` — every setup has a matched teardown on every exit path

- **area**: control flow, codegen RAII, FFI handles, runtime shutdown, concurrent test cleanup
- **invariant**: Allocations, opened handles, locks, mailboxes, profiler state, and partial select/join setups must be released on every exit path — return, panic, timeout, cancel, shutdown — with exactly one canonical release path per owned thing. Manual `close()` / `free()` and automatic drop must agree on who is canonical.
- **why**: Cleanup paths that only handle the happy case leak under failure; double-cleanup paths free twice. Both are silently catastrophic — the first as accumulating debt, the second as memory corruption.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `cleanup-all-exits` | Code stages setup work or mutates side-channel state before success is guaranteed, OR a checker/elaborator is built against a scaffold whose CFG is structurally flat (single basic block) and will widen later. | Evaluate return expressions before drops, reset pending state on failure, deactivate all intermediate loops, and clean partial select/join/ask setup before panicking, timing out, or returning early. When bounded teardown cannot wait for a live worker, transfer ownership to a reaper that performs the final join and free path; treat an outstanding join handle as the worker-liveness authority even if task state already says `Done`. For runtime-level reset: register every subsystem's cleanup in a session hook registry at init so it fires from every shutdown path. Check: when a Ready/Suspended task is cancelled via `mark_done(Cancelled)`, user-code Drop never fires (`task_scope.rs:627-644`); resources held in those stack frames leak until process exit. This is a known gap — actor terminate callbacks are the only WASM-safe cleanup path. Do not rely on scope cancel as a resource-release mechanism on any target. For IR-level move-checkers and drop elaborators: a forward-scan that walks a flat `MirStatement` stream is degenerate-correct only under a single-block CFG; the moment `if`/`match`/loop lowering produces real basic blocks, the checker must run as a per-block dataflow over a path-sensitive lattice (in v0.5 the four-state `Uninit | Live | Consumed | MaybeConsumed` lattice on `BindingId`) and drop plans must be enumerated per `Terminator::Return` block — not function-wide. `MaybeConsumed` at any Return exit on a `@linear` binding is a hard `MustConsume` rejection, not a "drop on some paths" recovery. | `cyclic-cfg-worklist-rpo` | A dataflow worklist pass is added or changed on a CFG that now contains back-edges (loop lowering, or any new `Terminator::Goto` that targets a lower-ID block). | Seed the worklist in Reverse Post-Order (RPO), not block-ID order. ### `ownership-over-locks` — fix lifetimes first, then add the smallest lock surface that remains

- **area**: runtime, concurrency, networking
- **invariant**: Shared-mutable-state correctness comes from clean ownership and causality, not from layered locks. When a protocol writer shares a stream, the lock surface must match the protocol's atomic unit (full frame), not the syscall.
- **why**: A broken lifetime model cannot be rescued by mutexes — at best the locks paper over the next mile of races. Mismatched lock granularity (syscall vs. protocol) admits interleaved bytes mid-frame, which is undebuggable from logs.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `ownership-over-locks` | Shared mutable state crosses threads, actors, or background workers. | Prefer `Arc`, atomics, owned routing tables, and mailbox-carried context, never pass stack pointers across threads, then add the smallest lock surface still required. | PR #784. |
| `split-framer-write-atomicity` | Multiple protocol framers or writers can write to the same socket, fd, or stream. | Identify every write-capable framer over a shared stream, acquire one per-connection operation gate around complete send/read-auto-flush/control-frame operations before any stream write, document lock ordering, and add deterministic short-write coverage that attempts a competing writer. If a close/drop path skips the gate, first remove sibling writers and document that invariant. | PR #1637 (closed #1632). |

## P1 — Parity, tests, diagnostics, and toolchain trust

### `parity-or-tracked-gap` — every user-facing surface ships in matching shape on every target, or names the gap

- **area**: native/wasm runtime, specs, stdlib surface, REPL/CLI/eval
- **invariant**: User-facing behaviour must match across native and WASM targets, and across CLI/REPL/eval entry points. Spec text and `pub fn` signatures must describe the shipped surface, not aspirational or ABI-typed names. Where parity cannot land yet, it must be a named tracked gap.
- **why**: Surface drift is the cheapest way to ship two compilers under one name. Users who learn the spec, then hit the runtime, then hit the second target should see the same shape every time.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `native-wasm-parity` | A runtime, scheduler, timer, channel, or actor behavior changes. | Implement both lanes or add a named TODO, add contract tests for timeout, cancel, and budget edges, and document intentional divergence where parity cannot land yet. For session-lifecycle hooks: `session_reset` must be called from both `hew_sched_shutdown` paths (`scheduler_wasm.rs` and `scheduler.rs`) so cleanup fires symmetrically. `task_scope` is absent on WASM today (`WASM-TODO(#1451)` at `task_scope.rs:782-783`); runtime cleanup paths that rely on scope cancel or structured concurrency must explicitly account for this gap — actor terminate callbacks are the only WASM-safe teardown path until #1451 closes. | PR #818/#787/#785/#754/#729; PR #1271; grounding doc 2026-05-08 async-cancel×closable investigation. |
| `spec-surface-alignment` | A spec example, API signature, or stdlib module uses a removed alias (`int`/`uint`) or mixes explicit widths inconsistently. | Use explicit-width integers (`i64`, `u64`, `isize`, `usize`) in all Hew source. `int`/`uint` are removed and produce a compile error with a suggestion. Verify examples against `hew-types/src/check/registration.rs`, `hew-types/src/check/methods.rs`, and the owning stdlib module; see `docs/stdlib-style-contract.md` for the full contract. | JOURNEY 2026-04-10 docs wire codec lane; Q56 alias removal. |

### `test-discipline` — tests assert the durable contract, run in any order, and trust their runner

- **area**: runtime tests, integration smoke tests, regen checks, runner CLI
- **invariant**: Tests must hold under the full suite in parallel (real isolation, not per-module locking that doesn't cross modules), assert the eventual stable invariant rather than a transient sequence, handshake on cross-process readiness, and run inside a runner that fails on parse errors and exposes timeout control.
- **why**: A test that only passes alone, only passes if the scheduler is fast enough, or hides discovery errors is worse than no test — it manufactures false green CI.

| id | trigger | apply | evidence |
| --- | --- | --- | --- |
| `global-test-isolation` | Tests share globals, phase state, supervisor state, or rely on execution order. | Add reset helpers and test-only serialization or mutexes where needed, run the full suite in parallel, and size the test to the actual race or nesting depth of the bug. When a test in module A touches globals owned by module B, expose a `pub(crate) static B_TEST_LOCK` and `pub(crate) fn reset_b_for_test()` from module B and acquire them in A before touching A's own lock; private per-module locks do not serialize across modules. Lock acquisition order must be consistent across all test sites that hold multiple locks simultaneously. | PR #1257; PR #1271. |
| `concurrent-test-final-invariant` | A concurrent or async test observes lifecycle state or cleanup across a race window. | Allow known valid intermediate states, require the eventual terminal state, and bind race windows to named scheduler or timeout constants at the assertion site. When a test dispatch must remain `Running` until a controlling function (drain, stop) has acted on the actor, use a release-flag spin loop (matching the `DRAIN_BUSY_LOOP_RELEASE` pattern) not a fixed sleep. | PR #813/#738/#751; PR #1484. |
| `network-smoke-readiness` | A smoke test launches a server and client or asserts teardown ordering across processes. | Wait for the server to bind or signal readiness before launching clients, loosen teardown assertions to the durable contract, and document intentionally unordered events. | PR #763. |

### `actor-surface-no-ask-verb` — actor request-reply is signature-driven, not keyword-driven

- **area**: language surface, actor model, spec review
- **invariant**: Hew's actor call surface is signature-driven dispatch. A `receive fn` returning `R` is a request-reply; one returning `()` is fire-and-forget. No caller-side keyword (`ask`, `.ask()`, `!`, `?`) is needed or permitted. `ask` is not lexer-recognised at any position in edition 2026; the select-arm actor discriminator is shape-based (`<actor>.<method>(<args>)` at HIR lowering), not a keyword (§4.11.1).
- **why**: Pre-2026 editions mirrored Akka classic (`.send()`/`.ask()`). The 2026 rewrite eliminated both: the checker derives tell-vs-ask from the callee's `receive fn` return type, and `ActorRef<T>` is already a distinct nominal type that marks the mailbox boundary. Reintroducing a verb would regress per-method type checking and break the signature-driven invariant without adding expressiveness.

### `structured-concurrency-scope-fork-split` — `scope` is the lifetime boundary; `fork` is the child-start verb

- **area**: language surface, structured concurrency, parser/HIR/codegen seam, spec review
- **invariant**: Hew's structured-concurrency surface is two non-synonymous keywords. `scope { ... }` opens the lexical-lifetime boundary; child tasks spawned inside are joined automatically at the closing brace. `fork name = call(...)` (or bare `fork call(...)`) inside a scope body is the only child-start verb; it is rejected outside a scope. The legacy `scope |s| { s.launch / s.spawn / s.cancel }` surface and the `fork { ... }` block form are both removed and must not be reintroduced; lifecycle hooks attach via actor-method attributes (e.g. `#[on(start)]` / `#[on(stop)]`), not via scope methods.
- **why**: Early Hew conflated the lifetime boundary with a lifecycle-method API (`scope |s|` with `s.launch / s.spawn / s.cancel`). Type-checking, HIR lowering, and C++ codegen all carried the cost: an extra parser state field, a desugar branch in `parse_expr`, three dead AST/IR variants, and a C++ MLIR op pair (`ScopeLaunchOp` / `ScopeCancelOp`) that only the legacy surface emitted. The 2026 ratification removed it entirely (no external consumers; in-tree migration). Reviving any part of it — a `fork { ... }` block form, an `s.launch / s.spawn` method, a unified `fork-is-also-scope` keyword — would re-conflate the two roles and reintroduce the seam.
